Maven Commands Guide

Distributed URL Shortener System • Operational Manual
Prerequisites: Ensure Java 17+ and Maven 3.8+ are installed and added to your system PATH.

1. Internal Tomcat (Development)

The project uses Spring Boot, which includes an Embedded Tomcat server. This is the easiest way to run the application during development or for simple deployments.

Run from Source (No Build Needed)

# Run the Dashboard/Backend directly mvn spring-boot:run

Run as Executable JAR

# First, clean and package the project mvn clean package -DskipTests

# Then run the resulting JAR file java -jar target/url-dashboard-1.0-SNAPSHOT.war

2. External Tomcat (Production)

For deployment to a standalone Apache Tomcat server (e.g., on a cloud VM or enterprise server), you must generate WAR files.

Generate WAR Files

# Clean previous builds and generate WARs for all modules mvn clean package -DskipTests

Deployment Steps

  1. Locate the .war files in the target folder (or deployment-artifacts folder).
  2. Copy the WAR file to your Tomcat's webapps directory.
  3. Start the Tomcat service (e.g., bin/startup.bat).
  4. Tomcat will automatically extract and deploy the application.

3. Common Maintenance Commands

Clean Project

Removes the target directory and all generated artifacts.

mvn clean

Install Dependencies

Downloads all dependencies defined in pom.xml to your local repository.

mvn install -DskipTests
