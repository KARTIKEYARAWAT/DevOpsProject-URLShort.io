Distributed URL Shortener System • Architectural Overview

1. Executive Summary

The Distributed URL Shortener System is a high-performance, scalable web application designed to solve the problem of handling massive volumes of URL redirection requests. Unlike traditional monolithic shorteners, this project employs a distributed architecture using Consistent Hashing to partition data across multiple storage nodes, ensuring high availability, fault tolerance, and horizontal scalability.

2. System Architecture

The Core Components

  • URL Router (The "Brain"): Acts as the entry point. It uses a consistent hashing ring to determine exactly which node is responsible for storing or retrieving a specific URL key. This eliminates the need for a central lookup table, removing a major bottleneck.
  • URL Nodes (The "Storage"): Independent, lightweight servers that store the actual key-value pairs in memory for sub-millisecond access latencies. Nodes can be added or removed dynamically.
  • URL Dashboard (The "Interface"): A modern, responsive React Application that visualizes the cluster state, allows users to shorten links, and provides real-time system metrics.

3. Key Technical Innovations

Consistent Hashing Algorithm

The system implements Consistent Hashing to distribute data. This ensures that when a new node is added or a node fails, only `1/N` of the keys need to be remapped, rather than the entire dataset. This is critical for elastic scalability.

Fibonacci Typography System

The user interface utilizes a custom design system based on the Fibonacci Sequence (13px, 21px, 34px, 55px). This mathematical approach to design creates a natural, harmonious visual hierarchy that is aesthetically pleasing and improves readability.
